# TAG:ON
## 동행 태그 기반 AI 스마트 글래스 안내 시뮬레이터

> 모든 글자를 읽는 것이 아니라, 지금 이동에 필요한 안내판만 골라 읽어준다.

## 1. 프로젝트 개요

TAG:ON은 시각장애인과 저시력자가 병원, 지하철역, 학교, 공공기관처럼 안내 정보가 많은 공간에서 필요한 안내판을 독립적으로 확인하도록 돕는 스마트 글래스 소프트웨어 콘셉트이다.

카메라 시야에 들어온 모든 문자를 OCR로 읽는 대신, 안내용으로 지정된 **D-TAG(동행 안내 태그)**가 부착된 안내판만 감지한다. 인식된 태그 ID를 안내 데이터와 연결하고, 현재 목적지와 관련된 핵심 정보만 방향·거리와 함께 음성으로 전달한다.

이 저장소의 `index.html`은 실제 스마트 글래스 하드웨어가 아니라 태그 기반 선택 안내의 효과를 비교·검증하기 위한 웹 기반 3D 시뮬레이터이다.

---

## 실행 환경

- `index.html`은 Three.js 라이브러리를 파일 내부에 포함한 **완전한 단일 HTML 파일**이다.
- 외부 CDN이나 별도 설치 없이 Chrome 또는 Edge에서 바로 실행할 수 있다.
- 음성 안내는 브라우저의 Web Speech API를 사용한다. 기기와 브라우저 설정에 따라 제공되는 한국어 음성이 다를 수 있다.
- 화면이 검게 보이거나 3D 초기화 오류가 발생하면 브라우저의 하드웨어 가속 설정을 확인한다.

## 2. 문제 정의

### 이동약자
- 시각장애인
- 저시력자
- 시야가 좁거나 복잡한 문자 정보를 빠르게 구분하기 어려운 사용자

### 대표 일상 상황
처음 방문한 병원에서 내과 접수처를 찾기 위해 여러 안내판을 확인하며 이동한다.

### 핵심 장벽
공간에는 안내판이 설치되어 있지만 다음 문제가 발생한다.

- 안내판의 존재와 위치를 시각적으로 확인하기 어렵다.
- 광고, 행사 안내, 시설 안내가 함께 있어 필요한 정보만 골라내기 어렵다.
- 모든 글자를 음성으로 읽으면 정보가 과도하게 전달된다.
- 방향과 거리 없이 문구만 읽으면 실제 이동 행동으로 연결하기 어렵다.
- 같은 안내가 반복되면 이동 집중도가 떨어진다.

### 해결할 문제

> 복잡한 공간에서 여러 문자 정보 중 현재 이동에 필요한 안내판만 선택하여 짧고 명확하게 전달한다.

## 3. 솔루션 정의

안내판에 부착된 D-TAG를 스마트 글래스 카메라가 인식하면 태그 ID에 연결된 정보를 조회한다. 현재 목적지와 관련성, 안내 우선순위, 사용자와의 거리, 시야 중심과의 가까움을 판단하여 하나의 안내를 선택한다.

### 일반 OCR과의 차이

| 일반 문자 인식 | TAG:ON |
|---|---|
| 시야의 모든 글자를 읽음 | D-TAG가 부착된 안내판만 읽음 |
| 광고와 안내 문구가 섞일 수 있음 | 이동 관련 정보만 선택 |
| 긴 문장을 그대로 전달 | 목적지·방향·거리 중심으로 요약 |
| 중요도 구분이 어려움 | 위험·비상·목적지 관련 안내 우선 |
| 같은 문구를 반복할 수 있음 | 반복 안내 지연 규칙 적용 |

## 4. D-TAG 설계

### 명칭
**D-TAG: 동행 안내 태그**

D는 다음 의미를 함께 가진다.

- Direction: 방향
- Destination: 목적지
- Donghaeng: 동행

### 기본 모양

- 정사각형 기반 고대비 마커
- 굵은 검은색 외곽선
- 왼쪽 위, 오른쪽 위, 왼쪽 아래의 기준점
- 중앙의 안내 종류 아이콘
- 하단의 태그 ID
- 색상은 보조 요소로만 사용하고 모양과 ID로 구분

### 안내 종류

| 종류 | 아이콘 | 예시 안내 |
|---|---:|---|
| 방향 | `→` | 오른쪽 12미터 앞에 내과 접수처가 있습니다. |
| 엘리베이터 | `↕` | 정면에 엘리베이터가 있습니다. |
| 화장실 | `WC` | 왼쪽에 화장실이 있습니다. |
| 접수·안내 | `i` | 정면 5미터 앞에 종합안내소가 있습니다. |
| 위험·통제 | `!` | 앞쪽 통로는 공사 중입니다. |
| 비상구 | 출구 아이콘 | 왼쪽에 비상구가 있습니다. |

### 부착 기준

- 안내판 좌측 또는 우측 상단
- 태그 상단이 항상 위를 향하도록 배치
- 태그 주변에 충분한 여백 확보
- 태그 ID와 안내 데이터는 1:1로 연결
- 실내 기준 약 8~12cm 정사각형을 가정

## 5. 시스템 작동 구조

### I-P-O

| 단계 | 내용 |
|---|---|
| Input | 카메라 영상, 현재 목적지, 음성 설정 |
| Process 1 | 화면 안에서 D-TAG 탐지 |
| Process 2 | 태그 ID와 방향 확인 |
| Process 3 | 태그 데이터 조회 |
| Process 4 | 목적지 관련성·거리·중앙 정렬·우선순위 평가 |
| Process 5 | 핵심 안내 문장 생성 및 반복 여부 확인 |
| Output | 음성 안내, 화면 자막, 진동 대체 표현, 기록 데이터 |

### 처리 흐름

```text
카메라 실행
  → 시야 안의 D-TAG 탐색
  → 태그 ID 인식
  → 안내 데이터 조회
  → 목적지 관련성과 우선순위 판단
  → 방향·거리 중심 문장 생성
  → 음성 및 화면 안내
```

## 6. 태그 인식 규칙

1. D-TAG가 없는 일반 글자는 TAG:ON 모드에서 읽지 않는다.
2. 태그가 시야 중심 근처에 일정 시간 이상 보일 때 안내한다.
3. 여러 태그가 보이면 목적지 관련성과 위험 우선순위를 먼저 고려한다.
4. 같은 태그는 설정된 시간 동안 반복하지 않는다.
5. 비상·위험 태그는 일반 목적지 안내보다 우선할 수 있다.
6. 인식 거리를 벗어나거나 시야에서 크게 벗어난 태그는 안내하지 않는다.
7. 사용자가 직접 다시 듣기를 선택할 수 있다.

## 7. 대표 시나리오

### 목표
병원 입구에서 출발하여 내과 접수처까지 이동한다.

### 공간 요소
- 병원 입구
- 광고 포스터와 건강검진 홍보판
- 종합안내소
- 화장실
- 엘리베이터
- 공사 중 통로
- 내과 접수처

### 체험 모드

#### 일반 OCR 모드
시야에 보이는 여러 안내판과 광고 문구를 함께 읽는다. 불필요한 안내 횟수와 정보 과부하를 확인한다.

#### TAG:ON 모드
D-TAG가 부착된 안내판만 인식한다. 목적지와 관련된 방향 안내와 위험 안내를 우선 전달한다.

### 비교 항목
- 목적지 도착 시간
- 잘못된 방향 선택 횟수
- 전체 안내 횟수
- 불필요한 안내 횟수
- 인식한 D-TAG 수
- 목적지 도착 성공 여부

## 8. 시뮬레이터 기능

### 필수 기능
- 타일 바닥·천장 패널·유리 출입구·접수 모니터·대기번호 표시기·대기 좌석·카페 진열장·엘리베이터·진료실 문·손 소독기가 포함된 현대적인 병원 공간
- 1인칭 이동과 마우스 포인터 위치 기반 시점 각도 직접 매핑
- 일반 OCR / TAG:ON 모드 전환
- D-TAG 시야 감지
- 안내판 앞면과 뒷면을 별도의 읽기 면으로 구성해 어느 방향에서도 글자가 거꾸로 보이지 않는 이중 안내판
- 목적지 관련 안내 우선순위
- 브라우저 TTS 음성 출력
- 반복 안내 방지
- 위험 안내 우선 처리
- 시작 전 프로젝트 목적·작동 방식·조작 방법 안내
- 목적지 도착 시 결과 안내, 종료 여부 확인, 계속 둘러보기와 다른 모드 재체험 선택
- 이동 기록과 결과 요약
- 미니맵과 목적지 표시
- `THREE.Bone` 기반 인물 계층과 로비·복도 이동 애니메이션


### 시야 체험 효과

3D 장면에만 교육용 저시력 효과를 적용한다. 상단의 `시야` 버튼으로 다음 모드를 순환한다.

- 저시력: 약한 흐림, 대비 저하, 채도 저하, 주변부 어두움
- 좁은 시야: 중앙 이외의 주변 시야 제한
- 눈부심: 밝은 빛 번짐과 낮은 대비
- 일반: 시야 효과 없음

이 효과는 특정 시각장애인의 실제 시야를 그대로 재현하지 않는다. 시각 경험은 원인과 개인에 따라 다르며, 정보 접근 장벽을 이해하기 위한 교육용 표현이다.

### 조작 방법
- `W` 또는 `↑`: 바라보는 방향으로 전진
- `S` 또는 `↓`: 후진
- `A`/`D` 또는 `←`/`→`: 좌우 이동
- 마우스 포인터: 포인터 위치를 시점 각도에 직접 매핑한다. 화면 중심은 정면, 좌우 끝은 약 ±90도, 상하 끝은 약 ±19도이며 포인터가 멈추면 시점도 멈춘다.
- `E`: 현재 시야 다시 스캔
- `R`: 마지막 안내 다시 듣기
- `M`: 음성 켜기/끄기
- 모드 버튼: 일반 OCR과 TAG:ON 전환

## 10. 성공 기준

### 기능 기준
- TAG:ON 모드에서 태그가 없는 일반 문구를 읽지 않는다.
- 태그 ID에 연결된 안내 문장이 정확히 출력된다.
- 목적지 관련 안내가 일반 시설 안내보다 우선한다.
- 위험 안내는 가까운 거리에서 높은 우선순위로 전달된다.
- 같은 안내가 짧은 시간에 반복되지 않는다.
- 목적지 도착 시 비교 결과가 표시된다.

### 사용자 경험 기준
- 안내 문장이 짧고 행동으로 연결된다.
- 방향과 거리 정보가 함께 제공된다.
- 정보 과부하가 줄어든다.
- 다시 듣기와 음성 끄기가 가능하다.
- 일반 OCR 모드보다 잘못된 이동과 불필요한 안내가 감소한다.

## 11. 실행 방법

1. `index.html`을 Chrome 또는 Edge에서 연다.
2. 시작 설명 화면에서 프로젝트 목적, 미션, 조작 방법을 확인한다.
3. `TAG:ON 체험 시작` 또는 `일반 OCR로 비교 시작`을 선택한다.
4. W/↑로 전진하고 S/↓로 후진하며, 마우스 포인터 위치로 시점을 직접 조절한다.
5. 내과 접수처에 도착하면 도착 메시지와 체험 기록을 확인한다.
6. 결과 안내창에서 시뮬레이션을 마치거나, 병원 공간을 계속 둘러보거나, 다른 모드로 다시 비교한다.

Three.js 라이브러리가 HTML 안에 포함되어 있어 별도 설치와 인터넷 연결 없이 실행할 수 있다.

로컬 파일 실행이 제한되는 환경에서는 간단한 로컬 서버를 사용할 수 있다.

```bash
python -m http.server 8000
```

브라우저에서 `http://localhost:8000/`을 연다.

## 11-1. 화면 및 상호작용 개선

- 마우스 포인터의 화면 위치를 카메라 각도에 직접 매핑한다. 좌우 약 ±90도, 상하 약 ±19도 범위로 제한되며 포인터가 멈추면 시점도 멈춘다.
- `W`/`↑`는 바라보는 방향으로 전진하고 `S`/`↓`는 후진한다. `A`/`D`와 좌우 방향키로 옆걸음 이동이 가능하다.
- 포세린 타일 바닥, 천장 패널과 라인 조명, 유리 출입구, 접수 모니터, 대기번호 표시기, 대기 좌석과 테이블, 카페 유리 진열장, 엘리베이터, 진료실 문, 손 소독기를 배치해 병원 공간을 고도화했다.
- 접수 직원, 앉아 있는 방문객, 로비와 복도를 이동하는 방문객은 `THREE.Bone` 계층으로 구성하고 팔·다리·머리 회전과 이동 애니메이션을 적용했다.
- 안내판은 얇은 판체의 앞면과 뒷면에 각각 별도의 읽기 면을 배치한다. 어느 방향에서 보더라도 한글·영문·D-TAG가 거울처럼 뒤집히지 않는다.
- 시작 안내창에는 프로젝트 목적, 일반 OCR과 TAG:ON의 차이, 미션과 조작 방법만 표시하며 구현 수정 내역은 노출하지 않는다.
- 내과 접수처에 도착하면 이동이 멈추고 도착 시간, 잘못된 이동, 전체 안내, 불필요 안내, 인식 태그 수가 표시된다.
- 결과 확인 후 시뮬레이션 종료, 계속 둘러보기, 다른 모드 재체험을 선택할 수 있으며 종료 후 처음 설명 화면이나 같은 모드 재시작으로 이동할 수 있다.

## 12. 한계와 확장 방향

현재 시뮬레이터의 태그 인식은 실제 카메라 영상 분석이 아니라 3D 공간의 위치·거리·시야 각도를 이용한 개념 검증 방식이다.

실제 제품으로 확장하려면 다음 요소가 필요하다.

- QR, ArUco, AprilTag 등 실제 머신비전 마커 인식
- 카메라 캘리브레이션과 거리 추정
- 실내 위치 측위
- 오프라인 태그 데이터 저장
- 개인정보와 촬영 영상 보호
- 안내판 설치·관리 표준
- 음성 외 햅틱 피드백
- 실제 시각장애인 사용성 검증
